# Jubako: プロジェクト固有の指示書

## プロジェクトの目的
Windows 標準のクリップボード履歴に似た操作感で、コピーした内容を階層構造（フォルダ）に整理し、永続化できるクリップボードマネージャーを開発する。

## 主要機能
- 階層化された永続化: クリップボードアイテムをフォルダに整理して保存。
- 履歴の永続化: 通常のクリップボード履歴も永続的に保持。
- ショートカット起動: `Win + Alt + V` で UI を最前面に表示。
- 直感的な UI: アイテムの選択、検索、管理が容易なインターフェース。

## 技術スタック
- 言語: Rust
- GUI フレームワーク: [iced](https://github.com/iced-rs/iced) - シンプルさと型安全性を重視した純粋な Rust 製 GUI ライブラリ。
- システム連携: `global-hotkey` (ホットキー監視), `arboard` (クリップボード操作), `rusqlite` (SQLite 永続化)。

## 実装戦略
1.  仕様書の確認を優先: コードを書く前に、必ず `docs/` ディレクトリ内の mdbook 仕様書を読み込み、要件を完全に理解すること。
2.  反復的な実装:
    *   ステップ 1: `iced` による基本 UI とウィンドウ挙動（フォーカス喪失で閉じる等）の実装。
    *   ステップ 2: クリップボードの変更監視と履歴保存の実装。
    *   ステップ 3: `Win + Alt + V` によるグローバルホットキーの実装。
    *   ステップ 4: SQLite または JSON による永続化レイヤーの実装。
    *   ステップ 5: UI への階層構造（フォルダ管理）の統合。

## 重要な制約事項
- UI パフォーマンス: `iced` のイベントループがホットキー監視やクリップボード監視を阻害しないよう、非同期タスクやサブスクリプションを適切に活用すること。
- ネイティブ連携: Windows API を直接利用する箇所（ポップアップ動作、タスクバー非表示、クリップボード変更通知）は、OS の挙動に忠実に実装すること。
- 仕様の遵守: `docs/src/` に記載された詳細仕様（レイアウト、スキーマ等）を「正」として扱うこと。
